1. User 👤

Represents:
customer
admin
restaurant owner

Typical fields:

id
name
email
password
phone
role

2. Restaurant 🍜

Represents a food shop/business.

Typical fields:

id
restaurant_name
address
phone
opening_hours
status
3. Menu Item / Food 🍔

Represents foods being sold.

Typical fields:

id
restaurant_id
name
description
price_mmk
image_url
stock/status

4. Category 📂

Groups menu items.

Example:

Drinks
Rice
BBQ
Dessert
5. Cart 🛒

Temporary selected items before ordering.

Typical fields:

user_id
total_price
created_at

6. Cart Item

Links foods inside cart.

Example:

Cart
  └── Burger x2
  └── Cola x1

Fields:

cart_id
menu_item_id
quantity
7. Order 📦 (VERY IMPORTANT)

Main business entity.

Represents finalized purchase.

Typical fields:

id
user_id
order_status
total_amount
payment_status
delivery_address
created_at
8. Order Item

Snapshot of purchased items.

Why separate from cart?

Because:

menu prices can change later
order history must remain correct

Fields:

order_id
menu_item_name
quantity
price_at_purchase

9. Payment 💳

Represents transaction/payment state.

Fields:

order_id
payment_method
amount
payment_status
10. Delivery (later stage)

Optional for MVP.

Fields:

driver
tracking
delivery_status
